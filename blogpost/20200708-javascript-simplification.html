Javascript simplification

Today I did something cool, or so I think anyway.

In essence what I did was to simplify and reduce the amount of code written to create the nav bar on the side and the header. I have previously treated all of these seperatly with each menuitem in the nav bar (July, My presence and so on) running it's own loop to generate parts of the navbar.

In the past I've used arrays to store the link information and the text information. Example "creating the blog" and "/blogpost/20200701-creating-the-blog.html" has been stored in seperate arrays, (together with the rest of the links under July). I have been meaning to simplify this for some time, but not really been sure how to sort the different varibles. Well today it came to me. It was almost to simple really. I decided to nest the arrays inside arrays then run a nested loop, the first going through the "container" array (the array that contains all the other arrays) and the nested loop running through the nestedarray. This took me from 7 almost identical loops (only variables differed) to one nested one. This uses 2 indexes a "mainIndex" and an "index" to keep track of the mainloop and nestedloop respectivly. This was not really on the planned todo, but kind of falls under simplifying the code. And wil lbe added today to the updated "website-plans" page.

But very basic and not really very secure I would assume, more for a dynamic approach to logging on atm and what content to show.
