Over the last year or so Cloudera and Dell Technologies UDS (Unstructured Data Storage division) have been validating solutions and products together. This has resulted in some reccomendations and certifications on CDH and HDP and hopefully soon CDP. This is fantastic news for customers and partners alike

Let me explain. About 18 months ago Cloudera and Hortonworks merged as companies, this means customers now have one less option on the market. There used to be 3 big players as in Cloudera, Hortonworks and Apache. The two former are commercial companies while Apache is opensource software.

Now I wrote that Cloudera and Hortonworks had merged and that this was good for the customers. And I mean to explain why here. It's simple really, instead of now having to get knowledge aobut 2 or 3 (potentially) more hadoop ecosystems there is now one single source which takes the best from the 2 largest. I have no idea if that will be for the best in 5 years or if hadoop will be the prefered way of analyising big data then, but for now it's good. Good for cloudera, good for Hortonworks, good for Dell and good for the customer.(OneFS also support other distributions like Ambari, Pivotal and others. How an PowerScale OneFS Hadoop implementation differs from a traditional Hadoop deployment

A Hadoop implementation with OneFS differs from a typical Hadoop implementation in the following ways:
  • The Hadoop compute and HDFS storage layers are on separate clusters instead of the same cluster.
  • Instead of storing data within a Hadoop distributed file system, the storage layer functionality is fulfilled by OneFS on an PowerScale cluster. Nodes on the PowerScale cluster function as both a NameNode and a DataNode.
  • The compute layer is established on a Hadoop compute cluster that is separate from the PowerScale cluster. The Hadoop MapReduce framework and its components are installed on the Hadoop compute cluster only.
  • Instead of a storage layer, HDFS is implemented on OneFS as a native, lightweight protocol layer between the PowerScale cluster and the Hadoop compute cluster. Clients from the Hadoop compute cluster connect over HDFS to access data on the PowerScale cluster.
  • In addition to HDFS, clients from the Hadoop compute cluster can connect to the PowerScale cluster over any protocol that OneFS supports such as NFS, SMB, FTP, and HTTP. PowerScale OneFS is the only non-standard implementation of HDFS offered that allows for multi-protocol access. PowerScale makes for an ideal alternative storage system to native HDFS by marrying HDFS services with enterprise-grade data management features.
  • Hadoop compute clients can connect to any node on the PowerScale cluster that functions as a NameNode instead of being routed by a single NameNode.
